## Codebase Patterns
- Next.js 14 App Router: all pages in src/app/, layouts in layout.tsx
- Tailwind custom colors: neon-green (#00ff41), neon-red (#ff2d2d), dark-navy (#0a0e1a), neon-orange, neon-yellow, neon-blue, neon-cyan
- Glass panel style via .glass-panel CSS class (defined in globals.css)
- Custom scrollbar, blink animation, and pulse-ring animation defined in globals.css
- node_modules was previously tracked in git — .gitignore was added and node_modules removed from tracking in Story 1
- Quality checks: `npx tsc --noEmit`, `npm run lint`, `npm test` (echo no tests)
- react-globe.gl requires SSR workaround (dynamic import with ssr:false) since it uses browser APIs
- All context/state in src/context/ThreatContext.tsx — single source of truth for selectedThreat, filters, filteredThreats, showNatoAssets
- All components in src/components/organisms/ (no atoms/molecules used)
- Mock data in src/data/ — threats.ts (12 zones), alerts.ts (25 alerts), military.ts (15 bases + 5 carriers)
- Globe uses combined pointsData array for both threats and NATO assets (discriminated by pointType field)
- Framer Motion used for: detail card slide-in (AnimatePresence), alert stagger, pill spring, stats fade-in, count-up
- prefers-reduced-motion respected via CSS media query in globals.css
- react-globe.gl ringColor callback types need explicit `(d: object)` annotation to avoid implicit any TS error
- AnimatePresence must wrap conditional renders for exit animations to fire in Framer Motion
